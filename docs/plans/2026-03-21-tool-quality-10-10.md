## Root Cause Analysis

### The `new` False Positive Chain

This is the #1 quality issue — it pollutes callees in `context`, `batch_context`, `neighborhood`, `callpath`, and `graph_export`.

**Chain:** `Vec::new()` in source → tree-sitter parses `scoped_identifier` with children `Vec` + `new` → `collect_body_refs` pushes scoped_identifier onto stack for descent (not a `crate::` path) → child `Vec` blocked by `NOISY_SYMBOL_NAMES` → child `new` passes through (removed from noisy list for constructor tracking) → `try_add("new", Call, None, None)` creates ref → `store_symbol_refs_fast` calls `SymbolIdMap::resolve("new", None, None)` → no impl-qualified, no file-qualified → name-only fallback finds `TreeRenderer::new` → stored with `confidence = "low"` → `get_callees` has **no confidence filter** → false positive rendered.

**Fix strategy:** Two-layer defense:
1. **Extraction layer** (Task 1): Extract type context from non-`crate::` scoped identifiers so `Database::new` resolves correctly and `Vec::new` gets filtered by the noisy check on the type part.
2. **Query layer** (Task 2): Add `confidence = 'high'` filter to `get_callees`, matching what `file_graph` already does.

### Constants in Call Graphs

`READY` and `STATUS_PATH` appear as "calls" in neighborhood trees. Root cause: tree-sitter classifies `identifier` nodes as `RefKind::Call` regardless of whether the referenced symbol is a function or constant. The `sr.kind = 'call'` filter on neighborhood queries matches these because the ref was stored as `call`. Fix: filter by **target symbol kind** — exclude `Const` and `Static` from call graph results.

Note: The `continue` needs to work with the loop structure. Since we're in a `for child in node.children(...)` inside a `while let Some(node) = stack.pop()` loop, the `continue` continues the inner for loop, which is correct — it skips `stack.push(child)`.

Wait — looking at the actual code structure, the match arms don't have a `stack.push(child)` after them. The `_ => { stack.push(child); }` is the default arm. So we need to restructure. The scoped_identifier arm currently does:

```rust
"scoped_identifier" => {
    if !col.handle_crate_path(&child, refs) {
        stack.push(child);
    }
}
```

Replace with:

```rust
"scoped_identifier" => {
    if col.handle_crate_path(&child, refs) {
        // crate:: path handled
    } else if let Some(context) = extract_scoped_context(&child, ctx.source) {
        // Type::method — use type as context
        let method_node = child.child(child.child_count() - 1);
        if let Some(mn) = method_node {
            let method_name = node_text(&mn, ctx.source);
            let line = i64::try_from(mn.start_position().row + 1).ok();

                    line,
```

```rust
/// Extract type context from a scoped identifier like `Database::new`.
/// Returns the type name if the pattern is `TypeOrIdent::method`.
fn extract_scoped_context(node: &Node, source: &str) -> Option<String> {
    if node.child_count() < 3 {
        return None;
    }
    let first = node.child(0)?;
    let kind = first.kind();
    if kind == "identifier" || kind == "type_identifier" {
        Some(node_text(&first, source))
    } else {
        None
    }
}

const CONSTRUCTOR_NAMES: &[&str] = &[
    "new", "from", "into", "default", "clone", "build", "init",
];

fn is_constructor_name(name: &str) -> bool {
    CONSTRUCTOR_NAMES.contains(&name)
}
```

```rust
pub fn handle_batch_context(
    db: &Database,
    symbols: &[String],
    full_body: bool,
    sections: Option<&[&str]>,
) -> Result<String, Box<dyn std::error::Error>> {
    if symbols.is_empty() {
        return Ok("No symbols provided.".to_string());
    }

    let mut output = String::new();
    for (i, symbol) in symbols.iter().enumerate() {
        if i > 0 {
            output.push_str("\n---\n\n");
        }
        let result = context::handle_context(
            db, symbol, full_body, None, sections, None, false,
        )?;
        output.push_str(&result);
    }

    Ok(output)
}
```

```rust
fn render_callers(
    db: &Database,
    output: &mut String,
    sym: &StoredSymbol,
    callers_path: Option<&str>,
    exclude_tests: bool,
) -> Result<(), Box<dyn std::error::Error>> {
    const MAX_CALLERS: usize = 30;

    let mut callers = db.get_callers(&sym.name, &sym.file_path, exclude_tests)?;
    if let Some(p) = callers_path {
        callers.retain(|c| c.file_path.starts_with(p));
    }
    if callers.is_empty() {
        return Ok(());
    }

    // Sort: production callers first, then test callers
    callers.sort_by_key(|c| c.is_test);

    let total = callers.len();
    let _ = writeln!(output, "### Called By\n");

    let mut shown_test_separator = false;
    for c in callers.iter().take(MAX_CALLERS) {
        if c.is_test && !shown_test_separator {
            let _ = writeln!(output);
            shown_test_separator = true;
        }
        let display = qualified_callee_name(c);
        let line = c.ref_line.unwrap_or(c.line_start);
        let _ = writeln!(output, "- {} ({}:{})", display, c.file_path, line);
    }
    if total > MAX_CALLERS {
        let _ = writeln!(
            output,
            "\n*({} more — use `references` for the full list)*",
            total - MAX_CALLERS
        );
    }
    let _ = writeln!(output);

    Ok(())
}
```
